## 目标
用户进入「心得社区」页面时，弹出一个选择窗口，让用户先选择自己关注的自律项目（戒烟 / 戒酒 / 戒奶茶 / 锻炼 / 戒淫 / 戒熬夜 / 全部 等），选择后自动进入对应分类的社区，过滤帖子并锁定发帖分类。

## 交互流程
1. 用户首次进入 `/community` 时，自动弹出一个居中的对话框（Dialog）。
2. 对话框标题：「选择你的自律项目」，副标题：「先告诉我们你想关注什么，进入对应的社区圈子」。
3. 列表展示所有自律项目卡片（含 emoji 图标 + 名称 + 简短描述），加一个「全部社区」选项。
4. 点击任一项 → 关闭弹窗 → 设置 `filter` 为对应分类 → 列表自动加载该分类的帖子（已有逻辑）。
5. 用户选择后，记忆到 `localStorage`（key: `community_default_category`），下次进入不再自动弹出，而是直接进入上次选择的圈子。
6. 在页面顶部新增一个「切换圈子」按钮（小图标 + 当前圈子名），点击可重新打开该弹窗以切换。

## 实现细节
- 文件：仅修改 `src/routes/community.tsx`
- 新增组件 `CategoryPickerDialog`，使用现有 `@/components/ui/dialog`。
- 分类来源：复用文件内已有的 `categories` 数组 + 「全部」选项；为每个分类配一个 emoji 映射（🚭 戒烟、🍺 戒酒、🧋 戒奶茶、🏃 锻炼、🧘 戒淫、🌙 戒熬夜、💡 心得、❓ 求助、🏁 里程碑、💪 鼓励、🌐 全部）。
- 状态：
  - `pickerOpen`：是否显示弹窗。
  - 初始化时读取 `localStorage`：若没有记录 → 弹窗打开、`filter` 暂为 `all`；若有记录 → 直接将 `filter` 设为该值，不弹。
- 选择回调：`setFilter(value)` + `localStorage.setItem(...)` + `setPickerOpen(false)`。
- 顶部 header 行右侧的「发布」按钮旁，加一个 `切换圈子` 按钮（ghost 风格，显示当前圈子标签）。

## 不改动
- 数据库、RLS、排行榜、发帖锁分类逻辑（`NewPostForm` 已支持 `lockCategory`，沿用即可）。
- 其他页面与组件不动。
